= Release Notes for v0.2.0
:toc:
:toclevels: 3
:sectnums:

Release Date: 2026-04-06

== FEATURES
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/4fa2f5b[4fa2f5b] Added new filtering flags to specify which resources are included in the generated import as well as if upstream dependencies should be included.  link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/8[#8]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/4fa2f5b[4fa2f5b] Added embedded reference strategy system (`reference`, `reference_with_fallback`, `variable`) for resolving UUIDs inside `jsonencode` node properties. DaVinci form UUIDs in flow `graph_data` are now emitted as Terraform input variables with human-readable names derived from the node title. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/8[#8]

== ENHANCEMENTS
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/65a8bea[65a8bea] `resource/pingone_davinci_flow_deploy`: Added support for import block generation. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/7[#7]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/65a8bea[65a8bea] `resource/pingone_davinci_flow`: Updated definition to keep empty values on required `js_links` child attributes. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/7[#7]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/65a8bea[65a8bea] `resource/pingone_davinci_flow`: Added `depends_on` block generation referencing `pingone_davinci_variable` resources used by the flow. Variable dependencies are discovered at runtime via the flow versions API and rendered as Terraform `depends_on` meta-arguments in both HCL and TF JSON output formats. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/7[#7]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/4fa2f5b[4fa2f5b] Fallback variables now carry a `Default` value (the original UUID) so that `--include-values` can populate `.tfvars` files with the raw ID. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/8[#8]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/4fa2f5b[4fa2f5b] PingOne form value is now a variable if found in `pingOneFormsConnector` properties to allow configuring value per environment. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/8[#8]

== BUG FIXES
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/65a8bea[65a8bea] Fixed `nil_value: keep_empty` not applying to nested attributes inside objects, lists, and sets. Attributes with this setting now correctly emit `""` at any nesting depth. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/7[#7]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/65a8bea[65a8bea] Fixed HCL template sequences (`${` and `%{`) in string values not being escaped in generated `.tfvars` files, which caused Terraform interpolation/directive errors. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/7[#7]

== NOTES
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/4fa2f5b[4fa2f5b] bump target PingOne Teraform Provider version 0.16.0-beta.2 => 0.18.0 link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/8[#8]
* link:https://github.com/pingidentity/pingcli-plugin-terraformer/commit/4fa2f5b[4fa2f5b] Updated README prerequisites: PingOne DaVinci export requires `DaVinci Admin Read Only` role. The `DaVinci Admin` role is recommended for full DaVinci Variable dependency resolution on flow resources. link:https://github.com/pingidentity/pingcli-plugin-terraformer/pull/8[#8]
